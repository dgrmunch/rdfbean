Use cases

General cases

  1. Given property has been defined for entity
  2. Given property is mapped to entity
  3. Given property has not been defined for entity
  4. Given property has been defined, but does not contain anything (is the case valid?)

Single vs. Multiple values cases

  1. Check if property contains multiple values (size)

Generic cases for multiple values

  1. Get a collection of all values
  2. Get a specific (by index) value from collection
  3. Remove all values
  4. Remove specific value
  5. Add a new value

Single valued cases (CRUD)

  1. Get default single value
  2. Get specific single value
  3. Add new value
  4. Update value?
  5. Remove specific value

Localized cases

  1. Check if value is localized
  2. Get default localized value
  3. Get specific localized value
  4. Get all localized values

Multivalued cases

  1. Get value count
  2. Get multiple values

Container cases

  1. Check if value is container
  2. Get multiple values

Special cases

  1. Get all properties including mapped properties
  2. Get all properties excluding mapped properties

Spring-binding

Tavoite: Tuottaa beanille editointi-sivu, jonka ominaisuudet ovat määritelty OWL-ontologialla. Ontologia määrittelee sen mitä ja miten beanin ominaisuuksia muokataan (olettaen että määritelmä on yhteensopiva beanin kanssa).

Mahdollistaa myös sellaisten ominaisuuksian muokkaamisen, joita ei ole suoraan mapattu luokka-ominaisuuksina, vaan "ekstra"-ominaisuuksina

Ongelmat

  1. Beanin yksilöivä avain (UID)?
  2. Yksilöllinen bindaus määritellyn propertyn ja arvon välillä
    - tehdäänkö muutokset suoraan beaniin vai Resourcen-kautta?
  3. Skeeman muutokset
  4. Ekstra-parametrien löytäminen ja bindaus
